# Skill: Ingest

## 触发方式
- `/ingest` — 处理 inbox/ 中所有文件
- `/ingest <file>` — 处理指定文件
- `/ingest <url>` — 抓取链接内容并处理

## 流程

### 1. 检查输入
- 列出 `inbox/` 中的所有文件
- 如果为空且没有提供 URL，告知用户没有待处理资料
- 支持三种输入：本地文件、URL 链接、粘贴的文本

### 2. 读取并分析
**本地文件：** 读取文件内容
**URL 链接：**
  1. 用 web_fetch 或浏览器工具抓取网页内容
  2. 如果是 PDF/arXiv 链接，提取文本内容
  3. 如果是视频链接（YouTube/B站），提取字幕/描述
  4. 保存原始网页内容到 `raw/主题/` 下（markdown 格式）
  5. 文件名用标题的 slug：`raw/decentralized-ai/clawcoin-arxiv-2604.19026.md`

**判断资料类型：** 论文/博客/视频/笔记/代码/推文

### 3. 归档原始资料
- 将文件从 `inbox/` 移动到 `raw/` 下对应主题子目录
- 如果主题目录不存在，创建它
- 文件名保持原样或用有意义的名称

```
例: inbox/clawcoin-paper.pdf → raw/decentralized-ai/clawcoin-paper.pdf
```

### 4. 生成 wiki 页面
- 在对应主题目录下创建 wiki 页面
- 用自己的话总结，不要简单复制原文
- 结构：
  ```markdown
  # 标题

  > 来源：[原始文件名](../raw/主题/文件名)
  > 日期：YYYY-MM-DD
  > 标签：#tag1 #tag2

  ## 一句话总结
  xxx

  ## 核心内容
  - 要点1
  - 要点2

  ## 关键细节
  ...

  ## 与其他知识的关系
  - 相关：[[已有页面A]]、[[已有页面B]]

  ## 我的思考
  ...
  ```

### 5. 建立链接
- 检查已有 wiki 页面，找到相关内容
- 在新页面中用 `[[页面名]]` 链接已有页面
- 在相关已有页面中补充指向新页面的链接

### 6. 完成确认
- 列出创建了哪些文件
- 列出了哪些关联链接
- 告知用户 inbox 已清空

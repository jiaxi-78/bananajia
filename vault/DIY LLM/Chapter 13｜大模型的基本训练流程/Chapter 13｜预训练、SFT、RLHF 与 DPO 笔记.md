# Chapter 13｜预训练、SFT、RLHF 与 DPO 笔记

> 按 Datawhale 第 13 章的主线整理大模型训练流程，并把容易卡住的点单独讲清楚：自回归、少量标注、JSONL、只算 assistant loss、PPO、DPO、权重插值、预算强制与思维标记。

## 第一组：先把整条训练流水线看清楚

这一章的主线其实很简单：**预训练让模型变聪明，SFT 让模型变得可用，RLHF / DPO 让模型更贴近人类偏好**。如果只记这句话，已经抓到了 80% 的框架。

预训练阶段面对的是海量无标注文本，目标仍然是最经典的 next-token prediction。模型通过“看上文预测下一个 token”学会语言规律、世界知识、代码模式和推理痕迹，但它此时更像一个很强的“续写器”，不一定知道什么叫“按用户要求回答”。

SFT 阶段不是重新给模型灌知识，而是把已有能力重新组织成更好用的行为：学会问答格式、指令跟随、角色扮演、工具调用、适当拒答，以及更稳定的回答风格。再往后，RLHF 或 DPO 处理的是偏好层：同样会答题，但什么样的答案更有帮助、更真实、更安全、更像人类真正想要的回答，需要进一步优化。

这也是为什么现在很多团队不再把三段流程看成完全孤立的三座山，而是更像连续训练链路：先纯预训练，再在预训练尾声混入高质量 SFT 数据，最后用偏好优化做更精细的行为对齐。

> **这一章最值得记住的主线**
> Pre-training 学知识和表示，SFT 学行为和格式，RLHF / DPO 学偏好与取舍。

## 第二组：预训练到底在做什么

### “在 5GB 的 BooksCorpus 上训练自回归语言模型” 到底是什么意思？

思考：这句话其实只包含两个动作。第一，BooksCorpus 是书籍文本数据集，5GB 指的是训练文本规模。第二，自回归语言模型（autoregressive language model）指的是：模型每一步都根据前面的 token 去预测下一个 token。

更直观地说，如果输入是 “I love eating”，模型的训练目标就是预测下一个词可能是 “pizza / sushi / apples”。训练时会把一整段文本切成很多“前缀 -> 下一个 token”的样本，所以标签并不是人工额外标出来的，而是文本自己就带着标签。

感想：这也是为什么预训练通常被归到自监督学习。它没有人工写好的标准答案，但能从原始文本中自动制造监督信号。模型不是在“背问答”，而是在“通过大量续写学习语言和知识”。

### 为什么说 GPT-3 是里程碑？什么叫“没有任何梯度更新”？

思考：GPT-3 证明了一件很反直觉的事：当模型足够大、预训练足够充分时，**只靠 prompt 里的上下文示例，就能临时适配任务**。这里说的“没有任何梯度更新”，意思是：推理时模型参数完全不变，没有反向传播、没有优化器、没有真正再训练。

比如你给它：

- `cat -> chat`
- `dog -> chien`
- `apple ->`

模型往往能直接补出 `pomme`。这不是它当场重新学习了法语，而是它把整段输入当成一个“还没写完的序列”，顺着这种模式继续生成。这种能力就是 in-context learning 或 few-shot learning。

感想：所以 GPT-3 的厉害之处，不只是参数大，而是第一次让大家真切看到：一个纯靠自回归预训练的大模型，能在不更新权重的前提下，仅凭上下文做出像任务理解一样的行为。

### 为什么 8B 模型还会训练到 36T tokens？

思考：这不是“8B 参数太小却喂了离谱多数据”，而是现代训练策略已经从“主要靠堆参数”转向“参数 + 更多数据 + 更久训练”共同驱动。36T tokens 也不意味着 36T 条完全独立的新知识，它可能包含重复暴露、多轮 epoch 和大量噪声数据。

更现实的理解是：参数量决定模型容量，token 数决定模型到底读了多少东西。一个 8B 模型并不“小”，但要把它喂饱、训练得稳定、从噪声数据里学出可靠模式，仍然可能需要非常大的 token 预算。

感想：这背后对应的其实就是 scaling laws。以前很多人直觉上觉得“更大参数 = 更强模型”，但近几年的实践越来越说明：**数据量、参数量、训练时长必须匹配**。

> **预训练的一句话总结**
> 预训练的本质不是教模型“怎么回答问题”，而是让它通过大规模续写学会语言结构、知识分布和表示能力。

## 第三组：SFT 为什么很重要，为什么又只需要少量标注

### 为什么预训练之后还要 SFT？为什么又是“少量标注”？

思考：预训练模型已经很会续写，但“会续写”不等于“会按用户要求回答”。SFT 的作用不是给模型补齐世界知识，而是把它已经学到的能力组织成更可控的行为：用户问问题，它知道自己在被要求回答；用户要求某种格式，它知道应该怎么组织输出。

所谓“少量标注”，是相对于预训练动辄几十亿、几百亿甚至上万亿 token 的规模来说的。因为大部分语言能力和知识已经在预训练中学过了，SFT 更像“行为校准”，所以几千到几万、几万到十几万条高质量样本就能显著改变模型表现。

感想：一个很有用的类比是：预训练像让人读了很多书，SFT 像教他“考试怎么作答”“客服怎么说话”“遇到危险请求怎么拒绝”。前者让模型变聪明，后者让模型变听话、变好用。

### Alpaca、ChatML、JSONL 分别是什么？

思考：Alpaca 是单轮指令格式，最典型的是 `instruction / input / output` 三元组；ChatML 更接近真实对话，把 system、user、assistant 等角色放进 `messages` 数组里，支持多轮对话。两者本质都是“给模型看人类示范答案”，区别主要在单轮还是多轮、是否显式区分角色。

JSONL（JSON Lines）不是一种新的语义结构，而是一种更适合训练管线的数据文件格式：**一行一个 JSON 样本**。它比普通 JSON 更适合大规模流式读取、并行切分、坏行容错和持续追加，所以训练数据常见的是 `.jsonl`，而不是外层套一个大数组的 `.json`。

感想：如果把 JSON 理解成“一个完整大文件”，那 JSONL 就是“很多条独立记录组成的数据流”。训练时一条条读、一条条 tokenize，再算 loss，会自然得多。

### 什么叫“只对 assistant 的 token 计算损失”？

思考：这句话的人话版是：**训练时只让模型学回答，不让它学提问**。user、system 这些消息仍然非常重要，因为它们作为上下文决定 assistant 应该说什么；但在 loss 里，真正被当作学习目标的通常只有 assistant 这部分。

例如一段对话是：

- User: `2+2 等于几？`
- Assistant: `等于 4`

这里 user 文本参与条件建模，但不参与目标误差；assistant 文本才是模型要尽量拟合的输出目标。否则模型会被同时训练去“生成用户问题”，这和对话系统的目标不一致。

感想：这个点第一次看确实容易懵，但一旦想清楚“user 是条件、assistant 是目标”，SFT 的训练逻辑就顺了。

### 为什么很多论文都说“高质量小数据”比“大量普通数据”更重要？

思考：SFT 和预训练最大的不同，是它对模型行为的塑形能力极强。预训练面对的是极大规模、弱监督的自然文本；SFT 面对的是高信号密度的人类示范答案。只要数据足够高质量、风格足够一致、任务边界足够清楚，即使数量少，也能强烈改变模型输出方式。

但反过来说，低质量 SFT 数据的伤害也会更直接：错误答案、格式污染、啰嗦风格、假引用、滥用拒答，都会被模型更快学进去。这也是为什么很多工作强调“少而精”，甚至用 1k、6k、9k 级别的精选样本就能追平或逼近更大的普通数据集。

感想：这里最反直觉但也最关键的一点是：**SFT 并不是越多越好，而是越能塑形，所以越要小心数据质量**。

> **SFT 的一句话总结**
> 预训练教模型“会什么”，SFT 教模型“怎么用这些能力回答你”。

## 第四组：SFT 后半段最容易混淆的几个概念

### 什么叫“权重插值”？

思考：权重插值就是把两个模型的参数按比例做线性混合。假设一个模型是“全量数据训练出的版本”，另一个模型是“高质量小数据训练出的版本”，那就可以做：

- `W_new = α * W_A + (1 - α) * W_B`

本质上是在参数空间里做折中，希望同时保留“大模型的广泛知识”和“小数据模型的好行为风格”。

感想：它不是重新训练，而是训练后的“模型融合”技巧。之所以很多时候有效，是因为神经网络参数空间并不总是离散断裂的，两个好模型之间的某些中间点也可能还是好模型。

### 什么叫“预算强制（budget forcing）”？

思考：这是推理阶段的控制技巧，不是训练阶段的损失函数。它的核心是：**给模型的思考过程设置 token 预算，不让它无限生成 reasoning**。尤其在思维链或 thinking model 里，如果不控制，模型很容易越想越长、成本越来越高，甚至越想越偏。

比如把 thinking 区间限制在 100 tokens 内，就是预算强制。它的意义不只是省钱，也是在推理质量和推理成本之间主动做权衡。

感想：这个概念之所以重要，是因为现在很多“强推理模型”的能力，不仅来自训练样本，也来自推理时的使用策略设计。

### 什么叫“插入思维标记（thinking markers）”？

思考：就是在训练数据里显式标出思考过程和最终答案，例如：

- `<thinking> ... </thinking>`
- `<answer> ... </answer>`

这样模型不只学习“直接回答”，还学习“先思考，再给答案”的组织方式。它等于把 reasoning 也变成监督信号的一部分。

但这里有个风险：模型可能学会的是“长得像 reasoning 的文本结构”，不一定真的学会了可靠推理。所以 thinking markers 不是魔法，它能帮助模型形成推理表面结构，也可能放大“看起来很有道理但其实是编的”这种现象。

感想：这也是为什么很多后续工作会继续引入奖励、验证、预算控制，甚至强化学习。单纯教模型写出思维过程，不等于它就一定会真实地思考。

### 什么叫“正则化策略的权衡”？

思考：如果直接用很多 SFT 数据猛训模型，模型可能会变得过度迎合这些数据风格，甚至出现灾难性遗忘，把预训练里学到的广泛能力覆盖掉。所谓“权衡正则化策略”，本质是在平衡两件事：一方面想让模型学到新的行为模式，另一方面又不希望它被这些数据“洗脑”得太彻底。

常见做法包括小学习率、早停、混入预训练数据、KL 约束、权重衰减等。现在越来越常见的思路，是在预训练后期就混入高质量 SFT 数据，也就是所谓 mid-training 或 decay-stage mixing，让知识学习和行为塑形更连续地发生。

感想：如果一句话概括，就是：**不是数据一多就应该狠狠干，而是要控制模型到底被后训练数据改成什么样子**。

## 第五组：RLHF 在干什么，为什么又需要奖励模型

### RLHF 的三阶段流程是什么？

思考：经典 RLHF 可以压缩成三步：

- 先做 SFT，得到一个会回答问题的基线策略模型。
- 再用偏好数据训练奖励模型，让它学会给“人更喜欢的回答”打更高分。
- 最后用 PPO 之类的强化学习方法，直接优化策略模型，让它在奖励模型定义的方向上变得更好。

这里的关键不是“模型能不能回答”，而是“面对多个都还行的回答，哪些更符合人类偏好”。这就需要额外的偏好信号，而不是只靠单条标准答案。

感想：SFT 更像“示范教学”，RLHF 更像“不断给分，然后让模型朝高分行为靠近”。

### 为什么奖励模型通常是“同底座 + 一个打分头”？

思考：因为奖励模型仍然需要理解 prompt 和回复的语义关系，所以通常直接复用 SFT 模型的底层表示能力，只把最顶层语言建模头换成一个线性打分头。这样做比从头训练一个全新评分器更高效，也能继承原本的大语言理解能力。

它看到的是同一 prompt 下的多个候选答案，人类或更强模型对这些答案做偏好比较，奖励模型则学习“更优答案分更高”。之后 PPO 就把这个标量分数当成优化方向。

感想：奖励模型不是在“回答问题”，它是在“给回答打偏好分”。这两个任务的角色不同，但底层语义理解往往可以共享。

### 幻觉、虚假引用和灾难性遗忘为什么会在这一阶段被反复提起？

思考：因为一旦模型开始强烈拟合特定格式，它就可能学到“形式上应该出现什么”，而不是“事实本身是否真的掌握”。例如训练数据里复杂概念后面总带引用，模型就可能学会“复杂内容后面应该跟个引用”，却不一定真的知道那个引用是否存在。

这就是为什么 John Schulman 会强调：**如果逼模型回答超出它知识范围的问题，本质上是在鼓励幻觉**。有时真正正确的标注反而应该是“我不知道”或“需要检索”。

感想：训练模型不只是“让它更会说”，还必须小心别把“为了填满响应格式而瞎编”一起强化出来。

> **RLHF 的一句话总结**
> 奖励模型负责定义“人类更喜欢什么”，PPO 负责把语言模型往这个方向推。

## 第六组：PPO 里最容易卡住的概念

### 什么叫“近似离线策略”，重要性采样又是在修什么？

思考：PPO 的一个工程核心是：**同一批采样出来的数据，希望能被反复用几次，而不是采样一次就扔掉**。但一旦模型已经更新过了，旧数据就不再完全匹配新策略，这就带来 off-policy 问题。

重要性采样通过一个概率比去修正这种分布偏移：

- `r_t = π_new(a|s) / π_old(a|s)`

如果新旧策略对这个动作的看法差不多，`r_t` 接近 1，旧数据仍然可信；如果差很多，权重就会显著偏离 1，说明这条旧数据对新策略的参考价值下降了。所谓“近似离线策略”，就是用这种修正过的旧数据，近似地支持多轮梯度更新。

感想：一句话说，重要性采样是在回答：**这条旧样本放到新策略眼里，还值不值得继续信**。

### “原始策略梯度”那个比值和优势是怎么用的？

思考：PPO 里的原始策略目标可以写成：

- `L = - r_t * A_t`

这里 `r_t` 是新旧策略对同一个动作概率的比值，`A_t` 是优势。如果 `A_t > 0`，说明这个动作比平均水平更好，那训练会鼓励新策略提高它的概率；如果 `A_t < 0`，说明这个动作更差，训练就会压低它的概率。

问题在于，`r_t` 如果变化太大，模型会在一次更新里被推得太狠，所以 PPO 才要引入 clip，把更新幅度限制在安全区间里。

感想：从直觉上看，这就是“好动作加权奖励，坏动作加权惩罚，但别奖惩过猛把策略推崩”。

### 什么叫 TD 误差？什么叫 GAE？

思考：TD 误差的直觉是：**我原本以为当前状态值多少，但真走一步之后，发现现实反馈和我预期不一样**。它衡量的是“一步之后，我对这一刻的估值到底低估了还是高估了”。

而 GAE（广义优势估计）是在 TD 误差基础上，把后续几步的信息也衰减加回来：

- TD 更短视、更稳定；
- GAE 看得更远、更接近真实长期收益，但波动也更大。

所以 `λ` 的作用就是在“偏差更大但更稳”和“更准但方差更高”之间做折中。PPO 里常常用 GAE 来估计优势，就是因为它在实践中能提供不错的稳定性和学习信号质量。

感想：如果用一句人话来记，TD 是“这一步比我预期好多少”，GAE 是“把后面几步也算进来后，这一步到底值不值”。

## 第七组：DPO 为什么会火，它和 PPO 真正差在哪

### DPO 的核心思想是什么？

思考：DPO 之所以受欢迎，是因为它把“偏好优化”从复杂的强化学习流程，重新压回成一种更像监督学习的训练方式。它不再先训奖励模型、再跑 PPO，而是直接拿一对偏好答案：

- `y_w`：更好的答案
- `y_l`：更差的答案

让模型学会：**相对于参考模型，我应该更像好答案、而不是差答案**。它本质是在比较概率比，而不是显式地学一个奖励函数。

感想：如果 PPO 像“先做裁判，再按裁判打分去训练选手”，那 DPO 更像“直接把正例和反例摆出来，让模型学谁更值得模仿”。

### SimPO 和长度归一化 DPO 在解决什么问题？

思考：原始 DPO 还需要一个参考模型。SimPO 的思路是把这层比较进一步简化，直接让好答案的概率比差答案高出一个 margin，从而省去参考模型、节省显存。

另一个现实问题是：模型和人类都常常偏爱更长的回答，于是“长文”可能因为总对数概率更大而天然占优势。长度归一化 DPO 就是把概率按 token 长度平均化，避免模型靠“写更长”而不是“答更好”来赢。

感想：这些变体都说明，偏好优化并不是一个静止公式，而是在不断修正训练中那些很实际的偏置来源。

### 这一章最后真正想提醒什么？

思考：不是“PPO 一定优于 DPO”或“DPO 一定更简单更好”，而是强化学习和偏好优化的结论往往非常依赖：基础模型、数据质量、奖励噪声、训练预算、KL 约束、长度偏差和实验环境。论文里的胜负，很多时候是条件性的。

感想：这也是这一章最后关于“过度优化”的提醒最重要的地方。把代理奖励优化得越来越高，不等于人类偏好真的越来越高。训练目标一旦和真实价值有偏差，优化越狠，跑偏可能越严重。

> **DPO 的一句话总结**
> DPO 用“好答案比差答案更该被偏爱”这个比较式监督目标，替代了 RLHF 里奖励模型 + PPO 的复杂闭环。

## 第八组：我自己要怎么记这一章

- 预训练：海量无标注文本，next-token prediction，本质是让模型学语言和知识。
- GPT-3：证明只靠大规模自回归预训练，也能出现 in-context learning。
- SFT：少量高质量标注，让模型从“会续写”变成“会回答、会听指令、会拒绝”。
- 数据格式：单轮常见 Alpaca，多轮常见 ChatML，文件格式常见 JSONL。
- 训练目标：user / system 是条件，assistant 才是主要的 loss 目标。
- 高质量数据：SFT 阶段质量常常比数量更重要，少量高质量样本就能显著改行为。
- RLHF：SFT -> 奖励模型 -> PPO，用偏好信号进一步调行为。
- PPO：重要性采样复用旧数据，clip 限制更新幅度，GAE 估计优势。
- DPO：不用奖励模型和 PPO，直接用偏好对做比较式优化。
- 现实工程：越来越多团队把高质量 SFT 数据混进预训练尾声，减少灾难性遗忘，也让训练链路更连续。

> **最后的总括**
> 这一章真正教的不是四个缩写，而是同一件事在不同阶段怎么被拆开做：先把模型做强，再把模型做可用，最后把模型做得更符合人类真正想要的行为。
